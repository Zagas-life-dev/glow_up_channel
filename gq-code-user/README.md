# QR Code User Profile Mini App

A mini app that allows users to generate QR codes for their profile and share contact information.

## Features

- ✅ **QR Code Generation** - Each user gets a unique QR code
- ✅ **Public Profile Page** - Displays user contact info when QR code is scanned
- ✅ **Save to Contacts** - One-click save to device contacts (vCard format)
- ✅ **Profile Customization** - Edit bio, phone number, and profile picture
- ✅ **QR Code Management** - Download, share, and manage your QR code

## Setup

### 1. Install Dependencies

```bash
cd gq-code-user
npm install
```

### 2. Environment Variables

Create a `.env.local` file:

```env
NEXT_PUBLIC_BACKEND_URL=http://localhost:8080
```

For production, set this to your backend API URL.

### 3. Backend Environment Variable

In the backend (`latest-glowup-channel/`), set the QR code base URL:

```env
QR_CODE_BASE_URL=http://localhost:3001
```

Or for production:
```env
QR_CODE_BASE_URL=https://your-qr-code-domain.com
```

### 4. Run Development Server

```bash
npm run dev
```

The app will run on `http://localhost:3001`

## API Endpoints

The app uses the following backend endpoints:

- `GET /api/qrcode/profile/:userId` - Get public profile (no auth)
- `GET /api/qrcode/my-qrcode` - Get user's QR code (auth required)
- `PUT /api/qrcode/profile` - Update QR profile (auth required)

## Usage

### For Users (Dashboard)

1. Navigate to `/dashboard` (or click "My QR Code" in main app)
2. View your QR code
3. Edit your profile:
   - Phone number
   - Bio (max 500 characters)
   - Profile picture URL
   - Enable/disable QR code
4. Download or share your QR code

### For Visitors (Scanning QR Code)

1. Scan the QR code
2. View the user's profile page at `/profile/{userId}`
3. Click "Save to Contacts" to save all details

## QR Code URL Format

The QR code generates a URL in this format:
```
{QR_CODE_BASE_URL}/profile/{userId}
```

Example: `http://localhost:3001/profile/507f1f77bcf86cd799439011`

## Authentication

The dashboard accepts authentication via:
1. URL parameter: `?token=...` (passed from main app)
2. localStorage: `accessToken` or `authToken`

## Notes

- The app runs on port 3001 by default to avoid conflicts with the main app (port 3000)
- Profile pictures should be publicly accessible URLs
- Phone numbers should be in international format for best compatibility
- Bio is limited to 500 characters
- QR code URL is generated by the backend using `QR_CODE_BASE_URL` environment variable

## Troubleshooting

### QR Code not loading
- Check that the backend is running
- Verify `NEXT_PUBLIC_BACKEND_URL` is set correctly
- Check browser console for errors

### Profile page not found
- Ensure `QR_CODE_BASE_URL` is set in backend environment
- Verify the URL format matches: `{QR_CODE_BASE_URL}/profile/{userId}`
- Check that the user has `qrCodeEnabled: true` in their profile

### Save to Contacts not working
- On mobile, ensure you're using HTTPS
- vCard download should work on all devices
- Check browser permissions for file downloads

### Authentication issues
- Verify token is being passed from main app
- Check localStorage for `accessToken`
- Ensure backend API is accessible
